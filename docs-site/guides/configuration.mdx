# Configuration Reference

Risoluto's configuration is layered with this merge order:

1. **Built-in defaults**
2. **WORKFLOW.md** (front-matter YAML)
3. **Persistent overlay** (`~/.risoluto/config/overlay.yaml`)
4. **Environment variables** and `$SECRET:name` resolution

## Managing Configuration

<Tabs>
  <Tab title="API">
    ```bash
    # View effective config
    curl -s http://127.0.0.1:4000/api/v1/config

    # View overlay only
    curl -s http://127.0.0.1:4000/api/v1/config/overlay

    # Update overlay
    curl -s -X PUT http://127.0.0.1:4000/api/v1/config/overlay \
      -H 'Content-Type: application/json' \
      -d '{"codex":{"model":"gpt-5.4"}}'

    # View config schema
    curl -s http://127.0.0.1:4000/api/v1/config/schema
    ```
  </Tab>
  <Tab title="Dashboard">
    Use the **System → Configuration** page in the dashboard to edit the overlay visually.
  </Tab>
</Tabs>

## Secrets Management

```bash
# Store a secret
curl -s -X POST http://127.0.0.1:4000/api/v1/secrets/SLACK_WEBHOOK_URL \
  -H 'Content-Type: application/json' \
  -d '{"value":"https://hooks.slack.com/services/..."}'

# List configured secrets
curl -s http://127.0.0.1:4000/api/v1/secrets

# Delete a secret
curl -s -X DELETE http://127.0.0.1:4000/api/v1/secrets/SLACK_WEBHOOK_URL
```

<Tip>
  Keep secrets out of committed workflow files. Use `$SECRET:name` references or environment variable expansion like `$LINEAR_API_KEY`.
</Tip>
