# API Endpoints

## Runtime & State

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/metrics` | Prometheus-format service metrics |
| `GET` | `/api/v1/runtime` | Runtime metadata — version, data dir, provider |
| `GET` | `/api/v1/state` | Full snapshot — queued, running, completed, token totals |
| `POST` | `/api/v1/refresh` | Trigger immediate reconciliation pass |
| `GET` | `/api/v1/transitions` | Available workflow transitions per issue |
| `GET` | `/api/v1/models` | List available Codex models from provider |

## Issues

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/api/v1/:identifier` | Issue detail with recent events and attempts |
| `GET` | `/api/v1/:identifier/attempts` | Archived attempts + current live attempt ID |
| `GET` | `/api/v1/attempts/:attempt_id` | Per-attempt event timeline |
| `POST` | `/api/v1/:identifier/model` | Save per-issue model override |
| `POST` | `/api/v1/:identifier/transition` | Transition issue to another workflow state |
| `POST` | `/api/v1/:identifier/abort` | Abort a running issue |
| `POST` | `/api/v1/:identifier/steer` | Inject steering message into running agent |

## Configuration

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/api/v1/config` | Effective merged operator config |
| `GET` | `/api/v1/config/schema` | Config schema and example payloads |
| `GET` | `/api/v1/config/overlay` | Persistent overlay values only |
| `PUT` | `/api/v1/config/overlay` | Apply an overlay patch |
| `PATCH` | `/api/v1/config/overlay/:path` | Set one overlay path |
| `DELETE` | `/api/v1/config/overlay/:path` | Remove one overlay path |
