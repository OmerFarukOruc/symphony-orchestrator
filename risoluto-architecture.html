Risoluto Architecture

Local orchestrator that turns Linear / GitHub issues into sandboxed Codex agent attempts and delivers the result as a GitHub pull request.

Runtime Flow (one attempt)

  • • 1. Orchestrator tick polls Linear / GitHub via Tracker Port
  • • 2. Dispatch sorts by priority; Workspace Manager preps a git worktree
  • • 3. Agent Runner spawns docker run with Codex CLI + config.toml
  • • 4. Codex calls OpenAI / tool providers over JSON-RPC / stdio
  • • 5. Git Manager pushes branch and opens / updates PR via GitHub API
  • • 6. AttemptStore persists turns, events, and RunOutcome to SQLite

Technology Stack

  • • Runtime: Node.js 22 · strict ESM TypeScript · pnpm
  • • Backend: Express 5 · chokidar watcher · node-cron
  • • Frontend: React 19 · Vite · SSE for live state
  • • Persistence: SQLite · Drizzle ORM · better-sqlite3
  • • Sandboxing: Docker (optional gVisor) · iptables egress
  • • Agent: OpenAI Codex CLI inside container
  • • Testing: Vitest · Playwright (E2E + visual)

Ports & Adapters

  • • OrchestratorPort → Orchestrator
  • • TrackerPort → Linear / GitHub adapters
  • • AttemptStorePort → SqliteAttemptStore
  • • ConfigOverlayPort → file + DB stores
  • • SecretsPort → SecretsStore (AES-256-GCM)
  • • TemplateStorePort → PromptTemplateStore
  • • AuditLoggerPort → AuditLogger
  • • GitIntegrationPort → GitManager

External Integrations

  • • Linear: GraphQL client · webhook registrar
  • • GitHub: REST issues · PR client · check runs · webhooks
  • • OpenAI / Codex: API key or OAuth (token refresh)
  • • Docker: local daemon · one sandbox per attempt
  • • Slack: outbound incoming webhook
  • • OS desktop: native notification bridge
  • • Prometheus: scrape /metrics for ops dashboards
